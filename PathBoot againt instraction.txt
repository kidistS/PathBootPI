System Architecture:
# The application will follow a layered architecture with the following layers:
# 1. Controller Layer: This layer will handle the incoming HTTP requests and route them to the appropriate service layer
     based on the domain of the question.
# 2. Service Layer: This layer will contain the business logic for processing the user input, handling translations, and
     generating responses using the LLM models. Each domain (Tax, Nav,Immigration) will have its own service class that
     implements the specific logic for that domain.
# 3. Data Access Layer: This layer will handle the interactions with the database for storing user interactions and responses.
     It will contain repository classes for managing the data persistence.
# 4. Model Layer: This layer will contain the data models for the application, such as the UserInteraction model for storing
     user interactions in the database and the Response model for representing the responses generated by the system.
# 5. Configuration Layer: This layer will contain the configuration classes for the application, such as the configuration for
     the LLM models and the database connection.
# 6. Enum Layer: This layer will contain the enums for the application, such as the Domain enum for representing the different
     domains (Tax, Nav, Immigration)
     and the Language enum for representing the different languages (English, Amharic, Norwegian).
# 7. Utility Layer: This layer will contain utility classes for common functionalities, such as a TranslationUtil class for
     handling translations between languages using the appropriate LLM models.
# 8. Session Management Layer: This layer will handle the session management for the application, keeping track of user
     interactions and providing a personalized experience. It will use Spring's session management features to manage user sessions
     effectively.
# 9. Database Layer: This layer will handle the interactions with the SQLite database for storing user interactions and responses.
     It will contain repository classes for managing data persistence and retrieval, ensuring that user interactions are stored securely
     and can be accessed for future reference and analysis.
# 10. Agent Layer: This layer will contain the agent classes for each domain (TaxAgent, NavAgent, ImmigrationAgent) that will implement
     the specific logic for processing user input and generating responses based on the domain context. Each agent will have its own
     prompt and data grounding file to ensure accurate and relevant responses for the respective domain.
